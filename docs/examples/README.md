# Examples

Examples show concrete Octopus workflows. They are not the product manual and
they are not default behavior.

Start with the core docs first:

- [Getting Started](../GETTING_STARTED.md)
- [User Guide](../USER_GUIDE.md)
- [Protocol Guide](../PROTOCOLS.md)
- [Operations Guide](../OPERATIONS.md)

Then use an example when you need a full scenario.

## Available Examples

| Example | Use it for |
| --- | --- |
| [Manufacturing intelligence](manufacturing-intelligence.md) | Customer-facing protocol walkthrough that creates an offline manufacturing intelligence command center artifact. |
| [Offline CSV analytics](offline-csv-analytics.md) | Browser-only multi-CSV analytics package walkthrough with local data handling. |

## Example Rules

- Keep customer scenario instructions inside the example.
- Do not make scenario-specific fields or files sound like product defaults.
- Do not use real private data unless the deployment and customer explicitly
  allow it.
- Validate examples through the Registry UI and real artifact actions, not
  private database edits.
