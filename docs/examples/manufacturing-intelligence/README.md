# Manufacturing Intelligence Walkthrough

This is a step-by-step example for building, publishing, running, and validating
a review-gated manufacturing intelligence protocol through the Octopus Registry
UI.

The customer in this walkthrough is the person using Octopus to build and run
the protocol. Manufacturing is the example domain; it is not a product default
and should not be hardcoded into the Registry, SDK, runtime, or generated
artifacts.

## What This Walkthrough Proves

By the end, a user has:

1. created a protocol from blank
2. declared output artifacts
3. assigned staged work to connected agents
4. built review loops
5. published the protocol
6. launched a real run
7. watched progress without using bot logs
8. opened the produced command center artifact
9. checked wide and narrow usability
10. exported/imported a protocol copy without overwriting the original
