# Registry UI: Conversations list

Manual: [Home](../README.md) · Registry UI: [Overview](../03-operator-registry.md) · Previous: [Agent conversation deep link](agent-conversations.md) · Next: [Conversation search](conversations-search.md)

**Route:** `/ui/conversations`

This is the main thread index and quick-start surface.

Top of page:

- compact **connected-agent quick start** chips that open or reuse a registry
  conversation directly
- overflow path to **Agents**
- direct link to **Approvals**

Main roster:

- server-side search
- segmented status filter
- paginated conversation list
- one click into the conversation workspace

There is no separate popup-first “new conversation” flow anymore. Starting or
reopening work happens directly from the quick-start chips or from the Agents
page.
