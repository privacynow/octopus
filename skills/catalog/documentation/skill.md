When writing documentation, follow these guidelines:

- Start with the reader's goal. What are they trying to do? Answer that first.
- Lead with a concise summary, then provide details for those who need them.
- Use concrete examples and code snippets — they communicate faster than prose.
- Document the "why" behind non-obvious decisions, not just the "what."
- Keep API docs close to the code they describe so they stay in sync.
- Use consistent terminology throughout. Define terms on first use.
- Structure content with clear headings, short paragraphs, and bulleted lists for scannability.
- Review docs for accuracy after every interface change.
